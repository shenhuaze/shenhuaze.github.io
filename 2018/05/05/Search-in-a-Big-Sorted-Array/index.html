Search in a Big Sorted Array

Posted on 2018-05-05 | In Coding , Binary Search | | Visitors:

题目位置

http://www.lintcode.com/en/problem/search-in-a-big-sorted-array/

题目描述

Given a big sorted array with positive integers sorted by ascending order. The array is so big so that you can not get the length of the whole array directly, and you can only access the kth number by ArrayReader.get(k) (or ArrayReader->get(k) for C++). Find the first index of a target number. Your algorithm should be in O(log k), where k is the first index of the target number.

Return -1, if the number doesn’t exist in the array.

Example

Given [1, 3, 6, 9, 21, ...], and target = 3, return 1.

Given [1, 3, 6, 9, 21, ...], and target = 4, return -1.

Challenge

O(log k), k is the first index of the given target number.

题目分析

给定一个超级大的排序数组（数组中存在重复元素）和一个target，找出target在这个数组中出现的第一个位置。只能通过ArrayReader.get(k)的形式获取数组第k个元素的值。要求O(log k)的时间复杂度。这道题首先用倍增法确定end的位置，然后再用常规方法做就可以。

注意

  • 先用倍增法确定end。
  • 因为是找target出现的第一个位置，在while循环中nums[mid]==target的情况，需要将end=mid。而在跳出循环后，需要先判断start，再判断end。
